# Documentation

This directory contains comprehensive documentation for the Digital Marketing Agent project.

## Structure

- **`architecture/`** - System architecture, design patterns, and technical decisions
- **`api/`** - API documentation, endpoints, and integration guides
- **`guides/`** - User guides, tutorials, and how-to documentation

## Documentation Standards

### Writing Guidelines

1. **Use Markdown** - All documentation in `.md` format
2. **Clear Structure** - Use headings, lists, and tables effectively
3. **Code Examples** - Include working code snippets with syntax highlighting
4. **Diagrams** - Use Mermaid for diagrams when possible
5. **Keep Updated** - Update docs with code changes

### Diagram Standards

- Use Mermaid for flowcharts, sequence diagrams, and architecture diagrams
- Include alternative text descriptions for accessibility
- Keep diagrams focused and readable

### API Documentation

- Document all endpoints, parameters, and responses
- Include request/response examples
- Document authentication and authorization
- Note rate limits and error codes

## Documentation Site

This project uses **MkDocs** for generating a beautiful documentation website.

To serve documentation locally:
```bash
mkdocs serve
```

To build static documentation:
```bash
mkdocs build
```
