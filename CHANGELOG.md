## Guidelines for Changelog Entries

### Categories
- **Added** - New features
- **Changed** - Changes in existing functionality
- **Deprecated** - Soon-to-be removed features
- **Removed** - Removed features
- **Fixed** - Bug fixes
- **Security** - Security fixes and improvements

### Format
- Group changes by category
- Use bullet points
- Start with a verb (Add, Change, Remove, Fix, etc.)
- Reference issue/PR numbers when applicable
- Keep entries concise but descriptive

### Example Entry
```markdown
### Added
- Lead capture API endpoint for digital ads integration (#123)
- Real-time analytics dashboard with conversion metrics (#124)

### Fixed
- Database connection pooling issue causing timeouts (#125)
```
